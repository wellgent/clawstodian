# clawstodian v0.4 brief - observable, cron-driven routines

Date: 2026-04-18

## Why this brief exists

v0.3 completed the catalog reorganization: specs moved into `programs/`, `HEARTBEAT.md` became a pure-prose coordinator, per-file symlinks collapsed to one directory symlink. The cutover was clean.

On the morning of 2026-04-18, an audit of the live `wellgent` install surfaced a harder problem. The gateway restarted at 22:48 UTC the previous night. Logs show `heartbeat: started` at 22:48:50 UTC and then zero tick events all day - no `sent`, no `skipped`, nothing. Cron at the same restart fired cleanly (Dreaming promotion at 09:04 UTC today). Today's daily note never got created. Untracked Dreaming outputs sat in the tree. `para-backfill` stayed disabled despite a pending queue. No signal reached the operator.

The specific failure is narrowly an OpenClaw question: heartbeat passes every potential tick through roughly seven silent-skip gates (`areHeartbeatsEnabled`, per-agent enablement, `isWithinActiveHours`, lane queues, delivery resolution, visibility) and any one of them can short-circuit without emitting. Cron has one observable delivery path per job; heartbeat has one gated delivery path total. The deeper issue is that the system had no way to notice. With `showOk: false`, isolated sessions, and no trace, a silent heartbeat failure is indistinguishable from a silent success. v0.4 exists to fix that, and to take the corrective far enough that the fix is architectural, not cosmetic.

## Tensions this iteration addresses

- **Unfalsifiable execution.** No per-tick fingerprint exists. The operator cannot distinguish "ran and found nothing" from "did not run."
- **Heartbeat-direct programs drop silently.** Pure-prose dispatch puts five programs behind one LLM decision each tick. A missed dispatch leaves no task record, no session, no log.
- **Overlapping routine semantics.** `para-tend` vs `para-backfill` in steady state, `workspace-tidiness` vs `git-hygiene` on ambient drift, `durable-insight` inline vs ambient triggers - all blurry.
- **Single-source drift.** `para_status` is used by two routines but missing from `templates/daily-note-structure.md`. `MEMORY.md` is referenced by routines but is not auto-loaded under `lightContext: true` in isolated sessions.
- **Install verification only checks structure.** Nothing verifies the machinery is delivering after the install lands.
- **"Programs" overloads the name.** OpenClaw uses "standing orders" for this anatomy. "Program" reads as larger and more monolithic than these actually are.

## v0.4 direction

### 1. Invert orchestration: cron-per-routine

Every routine becomes its own cron job. The heartbeat stops executing routines directly and becomes a pure orchestrator: read workspace state, decide which routines need attention, toggle their cron enabled state, nudge `--wake now` when needed, post an executive summary.

- Kill the `heartbeat-direct` execution class. Replace it with `heartbeat-toggled cron` (burst worker pattern generalized).
- Each routine now has its own session, task record, and delivery path. Failures isolate. Reruns are trivial.
- Restores the debuggability of `ops-daily` / `ops-para` / `ops-clean` without their orchestration sprawl. What those packages got right (cron with per-job notifications) was always sound; what they got wrong (hidden state, worker choreography) does not come back.
- Heartbeat cost per tick drops sharply. The orchestrator reads state and decides. It does not execute.

### 2. Mandatory observability, three layers

- **Tick trace.** Every heartbeat tick appends one line to `memory/heartbeat-trace.md`. Timestamp, state observed, routines toggled, summary posted. Append-only. Trivially greppable. Unfalsifiable.
- **Executive summary.** Every heartbeat tick posts a brief summary to the operator's logs channel: state of the workspace, which routines were triggered, any anomalies. Routine ticks where nothing changed still post a one-line "heartbeat OK, queues empty" so silence always means a scheduler failure, never a healthy tick.
- **Per-routine notifications.** Each cron routine announces its own completion summary to the logs channel. This is the `ops-daily` pattern that worked. Routine-level detail arrives routine-by-routine; heartbeat-level overview arrives tick-by-tick.

Three readers, three channels: trace for forensic debugging, executive summary for ambient awareness, per-routine notifications for detail on demand.

### 3. Rename: programs -> routines

- `programs/` -> `routines/`
- `docs/writing-a-program.md` -> `docs/writing-a-routine.md`
- `AGENTS-SECTION.md`, `HEARTBEAT-SECTION.md`, `README.md`, `CHANGELOG.md`, `INSTALL_FOR_AGENTS.md` updated.
- Workspace symlink becomes `clawstodian/routines -> ~/clawstodian/routines`.
- Cron `--message` updated to `"Read clawstodian/routines/<name>.md and execute."`
- Catalog language in `AGENTS-SECTION.md` shifts: "routines", not "programs".

"Routine" fits the nature: small, named, regularly-executed maintenance. Maps 1:1 to a cron job under the new model.

### 4. Install smoke test

Runs at the end of the install flow. Verifies the `AGENTS.md` section marker landed, the `HEARTBEAT.md` section marker landed, the `clawstodian/routines` symlink resolves, templates are present, cron entries registered. Pass/fail in under ten seconds.

A periodic workspace-audit routine ("is the machinery still delivering?") is deferred to a later iteration.

### 5. Consolidate to six routines

Collapse the nine-routine catalog to six, each owning one clear responsibility. Drop `health-sweep` as a standalone routine: runtime observability becomes a byproduct of the heartbeat orchestrator's executive summary (every tick inherently reads cron state, last-run times, and queue depths to decide what to do). Drop `durable-insight` as a first-class routine: its inline function folds into `daily-note`, its PARA-filing function folds into `para-extract`, and its unreliable "ambient" trigger goes away.

1. **daily-note** - Keep today's note current throughout the day. Append session activity (`sessions_list` + `sessions_history`), merge any `YYYY-MM-DD-slug.md` siblings inline into the canonical note, polish against conventions, update frontmatter. Subsumes `daily-notes-tend` and `durable-insight`'s inline-capture behavior. Fixes the "who merges today's slug files?" gap.

2. **seal-past-days** - Seal one past-day note per run: merge any remaining slug files, flip `status: active -> sealed`, set `para_status: pending`. Burst worker, heartbeat-toggled. Behaviorally unchanged from `close-of-day`.

3. **para-extract** - File entities from sealed notes into `projects/` / `areas/` / `resources/` / `archives/` per run. Flip `para_status: pending -> done`. One routine for the PARA filing responsibility. Replaces `para-tend`, `para-backfill`, and `durable-insight`'s ambient PARA filing. Burst worker, heartbeat-toggled.

4. **para-align** - PARA structural and semantic health:
   - Frontmatter integrity and `related:` pointer resolution.
   - Cross-reference updates after renames and moves (if `areas/people/jaros.md` becomes `areas/people/jaros-chimplie.md`, every referrer updates).
   - Naming and slug conventions (kebab-case, no spaces, consistent with `templates/para-structure.md`).
   - `MEMORY.md` dashboard currency as projects come and go.
   - `INDEX.md` coverage.

   Runs weekly by default; also triggerable by the heartbeat when `para-extract` reports drift mid-week.

5. **workspace-tidy** - Organize and clean: empty dirs, orphans, stale run-logs, ad-hoc files at wrong paths, archive-vs-delete decisions. Expanded from `workspace-tidiness` to cover active filing (moving a misplaced file to its intuitive home), not only removal.

6. **git-hygiene** - Commit meaningful drift stage-by-path, maintain `.gitignore`, write commit messages that tell the workspace story. Unchanged.

**Mapping from v0.3 to v0.4:**

- `daily-notes-tend` -> folded into `daily-note`.
- `close-of-day` -> renamed `seal-past-days` for clarity.
- `para-tend`, `para-backfill` -> consolidated into `para-extract`.
- `durable-insight` -> inline-capture folds into `daily-note`, filing folds into `para-extract`, ambient trigger dropped.
- `workspace-tidiness` -> renamed `workspace-tidy`, scope expanded to active filing.
- `weekly-para-align` -> renamed `para-align`, scope expanded (cross-references, naming, MEMORY.md).
- `health-sweep` -> dropped; folded into heartbeat orchestrator's executive summary.

### 6. Fix the single-source gaps

- Add `para_status` to `templates/daily-note-structure.md` frontmatter with documented values.
- Document the `MEMORY.md` + `lightContext: true` interaction in `docs/architecture.md` and flag routines that explicitly read it.

## Deferred

- **Periodic workspace-audit routine.** Install smoke-test covers install-time verification; a separate audit routine for ongoing delivery verification comes later.
- **Archive lifecycle policy.** When inactive projects or resources move from live PARA into `archives/` stays user-managed judgment; no routine automation in v0.4.
- **PARA sub-install separation.** Keep PARA coupled into the main install in v0.4. Revisit once v0.4 is stable.
- **Return to a deterministic `tasks:` block in HEARTBEAT.md.** The cron-per-routine inversion makes the pure-prose dispatcher's risk surface much smaller. If orchestrator cost or reliability becomes a concern, revisit.
- **Plugin or hook-based routines.** Out of scope.

## Out of scope

- New routine classes beyond heartbeat-toggled cron, demand-driven burst, fixed cron.
- Non-OpenClaw substrates.
- Any change to the workspace-as-ledger principle.

## Acceptance criteria

- A gateway restart that breaks heartbeat is discoverable by the operator within one tick via the logs channel.
- `memory/heartbeat-trace.md` shows every tick's outcome, greppable by date.
- Every routine run produces a task record, a session, and a summary to the logs channel.
- Install flow ends with a smoke-test pass report.
- The renamed `routines/` layout is consistent across every file that references it.
- `templates/daily-note-structure.md` documents every field real routines depend on.
- Routine count is six: `daily-note`, `seal-past-days`, `para-extract`, `para-align`, `workspace-tidy`, `git-hygiene`. Each has one clearly named responsibility.
