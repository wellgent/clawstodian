# git-hygiene

The workspace's git history tells a coherent story: meaningful drift is committed promptly with clear messages, `.gitignore` keeps ephemeral noise out, secrets never land in the tree, and the working tree is never left in a surprising state.

## References

- Related programs -> `clawstodian/programs/workspace-tidy.md`, `clawstodian/programs/daily-notes.md` (both produce changes this program commits)

## Conventions

- **Stage by exact path.** Never `git add -A` or `git add .`. A file is either explicitly committed or explicitly left alone.
- **One concern per commit.** Group dirty files into logical commits; do not batch unrelated changes.
- **Commit message shape**: `<topic>: <short description>`, lowercase, specific, meaningful. Common topics: `memory:`, `para:`, `config:`, `fix:`, `content:`, `insight:`, `guide:`, `skill:`, `ops:`, `docs:`.
- **Push immediately.** Unpushed commits are invisible to other sessions.
- **No AI attribution.** No `Co-Authored-By` referencing Claude, Anthropic, or agents. No `Generated by` footers.
- **No em-dashes in commit messages.** Use a regular hyphen-dash with spaces around it.

## Authority

- Stage specific files by exact path.
- Commit and push to branches the operator has opted into.
- Edit `.gitignore` to cover ephemeral or build-output patterns.
- Read `git log`, `git status`, `git diff`.

Must NOT force-push, rewrite history, amend published commits, reset --hard, or push to branches the operator has not opted into.

## Approval gates

- **Routine commits -> just commit.** A clean concern, a meaningful message, a known file.
- **Surface when unclear.** Unfamiliar file with commit/ignore/investigate ambiguity; a new extension or path pattern for `.gitignore`; anything that looks like a secret (`.env*`, keys, tokens, credentials).

## Escalation

- Tree is in an unexpected state (mid-rebase, detached HEAD, unresolved conflicts, untracked files from unknown source): surface and stop. Do not attempt to normalize.
- `git push` fails (network, auth, protected branch): surface the error and stop.
- A file that looks like a secret appears in the tree: surface; do not commit; do not ignore without operator direction.

## Exec safety

- Never `git add -A` or `git add .`. Stage by exact path.
- Never `--no-verify`. If a pre-commit hook fails, fix the underlying issue and commit again.
- Never `git reset --hard`, `git clean -f`, or force-push without explicit operator confirmation.
- Never rebase or amend published commits.

## Behaviors

### Commit drift

1. Run `git status`. If clean, nothing to do.
2. Group dirty files into logical commits, one concern per commit.
3. For each group: stage files by exact path. Commit with a `<topic>: <short description>` message per the conventions above.
4. Push immediately after each commit.
5. For any new untracked file that appears ephemeral (cache, build output, log): add to `.gitignore` and commit. If the file's nature is not obvious, surface instead of committing or ignoring.

## What NOT to do

- Never commit anything that looks like a secret; surface instead.
- No AI attribution lines in commit messages.
- No force-push.
- No `git reset --hard` without operator confirmation.
- No amending published commits.
- No committing to branches the operator has not opted into.
- No emoji in commit messages unless the workspace style already uses them.
